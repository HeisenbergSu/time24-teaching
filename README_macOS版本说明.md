# macOS 版本使用说明

## 🎯 三种 macOS 运行方式

### 方式一：Python 脚本版本（⭐ 推荐，最简单）

**适合对象**：所有 macOS 用户

**优点**：
- ✅ 无需打包环境
- ✅ 只需 Python 3（macOS 通常已预装）
- ✅ 体积小，启动快
- ✅ 易于更新

**使用方法**：
1. 解压 `macOS_Python版本` 文件夹
2. 双击 `启动系统.sh`
3. 首次运行会自动安装依赖和初始化数据库
4. 等待浏览器自动打开

**系统要求**：
- macOS 10.12+
- Python 3.6+（macOS 通常已预装）

---

### 方式二：GitHub Actions 自动打包（开发者）

**适合对象**：开发者，需要在云端自动打包

**使用方法**：
1. 将代码推送到 GitHub
2. 在 GitHub Actions 中手动触发：`Build macOS Version` → `Run workflow`
3. 或推送标签自动触发：`git tag v1.0.0 && git push origin v1.0.0`
4. 在 Actions 页面下载打包好的 macOS 可执行文件

**优点**：
- ✅ 无需本地 macOS 环境
- ✅ 自动化打包流程
- ✅ 适合 CI/CD

---

### 方式三：本地打包工具包（需要打包环境）

**适合对象**：有 macOS 打包环境的开发者

**使用方法**：
1. 将 `macOS_打包工具包` 复制到 macOS 系统
2. 在终端运行：`./在macOS上打包.sh`
3. 打包完成后，在工具包目录下会生成 `macOS` 文件夹

**系统要求**：
- macOS 10.12+
- Python 3.6+
- pip3
- PyInstaller（脚本会自动安装）

---

## 📋 对比表

| 特性 | Python 脚本版本 | GitHub Actions | 本地打包工具包 |
|------|----------------|----------------|----------------|
| 需要打包环境 | ❌ 不需要 | ❌ 不需要 | ✅ 需要 |
| 需要 macOS 系统 | ❌ 不需要 | ❌ 不需要 | ✅ 需要 |
| 体积大小 | 小 | 大（约50-100MB） | 大（约50-100MB） |
| 启动速度 | 快 | 慢 | 慢 |
| 独立性 | 需要 Python | 独立运行 | 独立运行 |
| 推荐度 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ |

---

## 💡 推荐选择

- **普通用户**：使用 Python 脚本版本（方式一）
- **开发者（有 GitHub）**：使用 GitHub Actions 自动打包（方式二）
- **开发者（本地 macOS）**：使用本地打包工具包（方式三）

---

## ❓ 常见问题

**Q: 如果 macOS 没有预装 Python 3 怎么办？**

A: 
- 访问 https://www.python.org/downloads/ 下载安装
- 或使用 Homebrew：`brew install python3`

**Q: Python 脚本版本和打包版本有什么区别？**

A:
- Python 脚本版本：需要 Python 环境，体积小，启动快，易于更新
- 打包版本：独立运行，体积大，启动慢，但不需要 Python 环境

**Q: 可以使用 GitHub Actions 自动打包吗？**

A: 可以！这是推荐给开发者的方式，无需本地 macOS 环境即可自动打包。
